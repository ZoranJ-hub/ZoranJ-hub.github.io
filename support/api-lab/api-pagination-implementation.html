API Pagination Implementation

Designed and implemented cursor-based pagination to deliver large datasets efficiently and reliably.

Challenge

An endpoint returned thousands of records in a single response, causing frontend performance issues and high bandwidth usage.

Solution

I implemented cursor-based pagination to return data in consistent chunks and avoid expensive OFFSET queries over large datasets.

Technical Deep Dive

  1. Evaluated existing queries and identified hotspots created by OFFSET/limit-based paging.
  2. Designed a cursor model using stable sort keys (e.g., created_at + id) to produce a reliable nextPageToken.
  3. Updated API contracts to include nextPageToken and prevPageToken and documented semantics.
  4. Implemented server-side query using WHERE (created_at, id) > (cursor_timestamp, cursor_id) pattern to page efficiently.
  5. Added tests to validate ordering, consistency, and boundary conditions.

Tools & Technologies

Node.js • Express • PostgreSQL • SQL (cursor queries) • Postman

Code Snippet Example

Express.js endpoint implementing cursor-based pagination with PostgreSQL.

// GET /api/items?limit=50&cursor=BASE64
app.get('/api/items', async (req, res) => {
  const limit = Math.min(parseInt(req.query.limit) || 50, 100);
  const cursor = req.query.cursor ? JSON.parse(Buffer.from(req.query.cursor, 'base64').toString()) : null;

  let rows;
  if (cursor) {
    // cursor example { created_at: "2025-01-01T00:00:00Z", id: 123 }
    rows = await db.query(
      'SELECT * FROM items WHERE (created_at, id) > ($1, $2) ORDER BY created_at, id LIMIT $3',
      [cursor.created_at, cursor.id, limit]
    );
  } else {
    rows = await db.query('SELECT * FROM items ORDER BY created_at, id LIMIT $1', [limit]);
  }

  const items = rows.rows;
  const nextCursor = items.length ? Buffer.from(JSON.stringify({ created_at: items[items.length-1].created_at, id: items[items.length-1].id })).toString('base64') : null;

  res.json({ items, nextPageToken: nextCursor });
});

SQL Example

// PostgreSQL cursor-style query
SELECT id, created_at, payload
FROM items
WHERE (created_at, id) > ($1::timestamptz, $2::bigint)
ORDER BY created_at, id
LIMIT $3;
