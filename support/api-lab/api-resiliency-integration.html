API Resiliency Integration

Implemented circuit breaker patterns to prevent cascading failures and provide graceful fallback behavior.

Challenge

The application experienced full outages when downstream services were slow or unresponsive, resulting in poor user experience and a spike in support tickets.

Solution

I implemented a Circuit Breaker pattern using Resilience4j to detect failure conditions, fail fast, and provide fallback responses while preserving system stability and recovery.

Technical Deep Dive

  1. Analyzed failure modes and SLA expectations for downstream services.
  2. Introduced Resilience4j circuit breaker around the client calls with tuned thresholds for failure rate and wait duration.
  3. Defined fallback handlers returning cached or degraded responses where appropriate.
  4. Implemented health metrics and monitoring (Prometheus + Grafana) to observe breaker state transitions.
  5. Used canary deployments and staged rollouts via CI/CD to validate behavior under load.

Tools & Technologies

Java • Spring Boot • Resilience4j • Maven • Prometheus • Grafana

Code Snippet Example

Spring Boot controller with Resilience4j circuit breaker and fallback handler.

// pom.xml dependency
// <dependency>
//   <groupId>io.github.resilience4j</groupId>
//   <artifactId>resilience4j-spring-boot2</artifactId>
//   <version>1.7.1</version>
// </dependency>

@RestController
public class OrdersController {

  private final OrderService orderService;

  public OrdersController(OrderService orderService) {
    this.orderService = orderService;
  }

  @GetMapping("/api/orders/{id}")
  @CircuitBreaker(name = "ordersService", fallbackMethod = "ordersFallback")
  public ResponseEntity<Order> getOrder(@PathVariable String id) {
    return ResponseEntity.ok(orderService.fetchOrder(id));
  }

  public ResponseEntity<Order> ordersFallback(String id, Throwable t) {
    // return cached or degraded response
    Order cached = orderService.getCachedOrder(id);
    return ResponseEntity.status(HttpStatus.SERVICE_UNAVAILABLE).body(cached);
  }
}

Circuit Breaker Configuration (example)

resilience4j.circuitbreaker.instances.ordersService.registerHealthIndicator=true
resilience4j.circuitbreaker.instances.ordersService.slidingWindowType=COUNT_BASED
resilience4j.circuitbreaker.instances.ordersService.slidingWindowSize=20
resilience4j.circuitbreaker.instances.ordersService.failureRateThreshold=50
resilience4j.circuitbreaker.instances.ordersService.waitDurationInOpenState=30s
